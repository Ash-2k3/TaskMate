<decisions>
## Implementation Decisions

### Font system
- **D-01:** Install `@fontsource/inter` (offline-safe, no CDN) — `npm install @fontsource/inter`
- **D-02:** Import weights 400, 500, 600 only in `src/renderer/index.css` — do NOT import 300 (renders poorly in Electron's Chromium)
- **D-03:** Set font-feature-settings globally: `'cv11' 1, 'tnum' 1, 'liga' 1, 'calt' 1` — cv11 = single-story 'a', tnum = tabular numbers for stats alignment
- **D-04:** Add `-webkit-font-smoothing: antialiased` and `-moz-osx-font-smoothing: grayscale` to `:root` in index.css
- **D-05:** Extend `tailwind.config.js` `fontFamily.sans` to `['Inter', 'system-ui', '-apple-system', 'BlinkMacSystemFont', 'sans-serif']`
- **D-06:** Add two custom Tailwind font sizes: `'2xs': ['11px', { lineHeight: '1rem' }]` for section labels, `'ui': ['13px', { lineHeight: '1.25rem' }]` for meta/secondary text

### Type scale — 3 levels
- **D-07:** Screen title (Today, Reflections, Summary, Settings): `text-2xl font-semibold` — already correct in most screens, verify all
- **D-08:** Section subheading (week date in WeeklySummary): `text-base font-medium` — currently `text-lg`, reduce
- **D-09:** Section label (uppercase caps in WeeklySummary): `text-2xs font-semibold uppercase tracking-wider text-muted-foreground` — currently `text-sm` uppercase, reduce to 11px
- **D-10:** Task title / body text: `text-sm font-normal` (14px) — already correct in TaskRow
- **D-11:** Meta / label / date / stats: `text-ui` (13px) — currently `text-sm` in several places (TodayView date subline, Settings stats, nav labels, ReflectionModal question labels)
- **D-12:** Overdue badge / helper text: `text-xs` (12px) — already correct

### Color tokens — warmer foreground
- **D-13:** Update `--foreground` in `:root` from `222.2 84% 4.9%` to `220 15% 8%` — warmer near-black, less blue-tinted, matches Linear/Notion philosophy
- **D-14:** Update dark mode `--border` from `217.2 32.6% 17.5%` to `217.2 32.6% 22%` — currently same as `--muted`, borders are invisible in dark mode; fix makes them subtly visible
- **D-15:** All other color tokens unchanged — indigo two-touchpoint discipline (Add Task + active nav) preserved from Phase 04.1

### Checkbox hover affordance
- **D-16:** Checkbox button (TaskRow completion circle): add `hover:border-primary hover:bg-primary/10` — communicates interactivity on hover using indigo in a transient (hover-only) context; not a persistent third touchpoint
- **D-17:** Completed state checkbox: no change — already uses strikethrough + muted color

### Nav bar grounding
- **D-18:** Nav bar background: change `bg-background` to `bg-card` — creates subtle visual panel distinction from content; `bg-card` is the standard shadcn token for elevated surfaces
- **D-19:** Nav label text: change `text-sm font-medium` to `text-ui font-medium` (13px) — nav labels were matching body text size; reduce to secondary size
- **D-20:** Nav bar border stays `border-t border-border` — no change

### TodayView fixes
- **D-21:** Header bottom: change `<div className="flex items-start justify-between px-6 py-6">` to `px-6 pt-6 pb-4 border-b border-border` — grounds header visually, separates from task list
- **D-22:** Date subline spacing: change `mt-0.5` to `mt-1` — too tight currently
- **D-23:** Date subline size: change `text-sm text-muted-foreground` to `text-ui text-muted-foreground` (13px meta text)
- **D-24:** Settings gear button touch target: add `p-1 rounded` to the gear icon button wrapper — icon buttons need a minimum touch/click target

### EmptyState fix (critical)
- **D-25:** Replace `flex flex-col items-center justify-center h-64` with `px-4 py-8` — flush-left, matching TaskRow's `px-4` baseline; centered empty state breaks layout language of flush-left app
- **D-26:** Remove `<h2 className="text-lg font-semibold">` heading from EmptyState — quiet empty states don't shout; replace with two `text-sm text-muted-foreground` lines
- **D-27:** EmptyState first line: `"All clear — nothing left for today."` (text-sm muted)
- **D-28:** EmptyState second line: `"Add a task to get started."` (text-ui muted, mt-1)

### WeeklySummary fixes
- **D-29:** Stat rows spacing: change `space-y-1` to `space-y-2` in the "This week" section — 4px is too tight for 14px text
- **D-30:** All 3 section headers: change to `text-2xs font-semibold uppercase tracking-wider text-muted-foreground mb-3`
- **D-31:** "Week of" date: change from `text-lg font-medium` to `text-base font-medium` — it's a data label, not a screen title
- **D-32:** Past-week selector `<select>`: ensure it inherits Inter font (native `<select>` may not inherit font — add `font-sans text-ui` explicitly)

### ReflectionsHistory fixes
- **D-33:** Outer wrapper: change `p-6` to `px-0 pt-6` — current `p-6` gives `h1` a 24px indent but rows use `px-4`, causing misaligned left edges between header and row text
- **D-34:** Apply `px-6` directly to `<h1>` element so header text aligns with other screen headers
- **D-35:** Header margin: change `mb-4` to `mb-6` — matches other screens' header-to-content spacing

### Settings fixes
- **D-36:** Stats text: change `text-sm` to `text-ui` (13px) in the record counts section — secondary data should use secondary text size
- **D-37:** Add `border-t border-border pt-6` before the "Delete all data" section — visual separation signals the destructive zone

### ReflectionModal fixes
- **D-38:** Question label text: change `text-sm font-medium` on question labels to `text-ui font-medium` (13px) — label should be lighter than textarea content

### Back navigation button (AddTask, EditTask, Settings, ReflectionsHistory)
- **D-39:** Back button pattern: change to `flex items-center gap-1 text-ui text-muted-foreground hover:text-foreground mb-4` with `<span className="text-base leading-none">←</span><span>Today</span>` — arrow slightly larger than label text, intentional hierarchy

### Preserved from Phase 04.1 (do not touch)
- `bg-background` task rows (no card fill)
- `hover:bg-muted/40` row hover
- `border-b border-border` row separators
- Weight-only priority signal
- Indigo on Add Task button + active nav tab only (persistent color)
- Flush-left forms (`px-6 py-6`, full-width)
- `focus-visible:ring-1` input pattern
- No `transition`, `duration`, `ease`, or `animate-*` classes anywhere

### Claude's Discretion
- Exact `text-2xs` usage vs `text-[11px]` — use whichever is available after tailwind.config.js changes
- Exact ordering of CSS imports in index.css
- Whether to add `font-sans` class to `<html>` or `<body>` vs rely on Tailwind's base reset

</decisions>

<specifics>
## Specific Ideas

- "Structure should be felt, not seen" (Linear's philosophy) — this guides the whole phase: soften, don't remove structure
- Inter `cv11` is the same feature used in Tailwind Catalyst (Tailwind Labs' official premium component kit) — not a custom choice
- The EmptyState center-alignment is the single most visually jarring inconsistency currently. Fix it first.
- Things 3-style checkbox hover: the affordance signal on hover eliminates the "is this a button?" confusion without animation
- `bg-card` nav feels like Things 3's tab bar — grounded, not floating

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- All existing shadcn components (`button.tsx`, `input.tsx`, `dialog.tsx`, etc.) inherit font via `font-sans` — Inter upgrade is automatic once tailwind.config.js is updated
- CSS variable tokens (`--foreground`, `--border`, `--muted`, `--card`) are used consistently; changing them in `index.css` cascades everywhere

### Established Patterns
- `text-sm` = 14px body text (keep for task titles, main content)
- `text-xs` = 12px for badges/helpers (keep)
- `text-[13px]` → replace with `text-ui` after config change
- `text-[11px]` → replace with `text-2xs` after config change
- No `transition` class anywhere — preserve this constraint

### Integration Points
- `tailwind.config.js` changes affect all files — do this first in Wave 1
- `index.css` changes (font imports, tokens) must come before screen changes
- TaskRow checkbox change is isolated to `src/renderer/components/TaskRow.tsx`

</code_context>
