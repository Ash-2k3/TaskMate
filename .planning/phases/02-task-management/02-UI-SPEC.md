| Token | Value | Usage |

## Typography

All sizes use `font-family: system-ui` (OS native stack). Two weights only.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 18px | 600 (semibold) | 1.2 |
| Display | 24px | 600 (semibold) | 1.2 |

Usage map:
- **Display (24px/600):** Screen title ("Today" on Today view, "Add Task", "Edit Task")
- **Heading (18px/600):** High-priority task titles (bold treatment required by spec)
- **Body (14px/400):** Standard task titles (Medium and Low priority), input field text, date picker values
- **Label (12px/400):** Overdue badge text ("N days ago"), priority selector option labels, helper text under inputs

## Color

System-adaptive: shadcn CSS variables resolve to light or dark values based on OS `prefers-color-scheme`. No manual theme toggle in Phase 2.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` — `--background` | `hsl(222.2 84% 4.9%)` — `--background` | Screen background, window chrome |
| Secondary (30%) | `hsl(210 40% 96.1%)` — `--muted` | `hsl(217.2 32.6% 17.5%)` — `--muted` | Task row surface, Add Task form card |
| Accent (10%) | `#6366f1` — mapped to `--primary` | `#6366f1` — mapped to `--primary` | See reserved list below |
| Destructive | `hsl(0 72.2% 50.6%)` — `--destructive` | `hsl(0 62.8% 30.6%)` — `--destructive` | Delete task button only |

**Accent (#6366f1 / --primary) reserved for:**
1. High-priority task left border (3px solid `--primary`)
2. Save button background (primary action on Add Task and Edit Task screens)
3. "+ Add Task" button background on Today view
4. Focus ring on active inputs (shadcn default, uses `--ring` mapped to `--primary`)

Accent is NOT used for: task row hover states, overdue badges, priority labels, or any decorative purpose.

**Overdue badge color:** `--muted-foreground` (grey text, `--muted` background). Not red. Calm, informational tone.

**Low-priority opacity:** Task row rendered at `opacity: 0.6` — applies to title text and all row content. No color change.

## Screen Specifications

### Screen 1: Today View

**Layout:** Single-column, full-height. Fixed header, scrollable task list body.

**Header:**
- Display text: "Today" (24px/600)
- Sub-label: current date in format "Friday, March 21" (14px/400, `--muted-foreground`)
- "+ Add Task" button: top-right, indigo (`--primary`) background, white label, 14px/400, `sm` padding (8px vertical, 16px horizontal)

**Task List:**
- Maximum 7 task rows displayed (TASK-05, Miller's Law constraint)
- Rows sorted ascending by due date; tasks with no due date appear after dated tasks
- Row height: minimum 48px (md padding 16px top + 16px bottom on 14px text)
- Row surface: `--muted` background, `--border` bottom separator (1px)
- No hover animation — hover state: `--accent` background only (instant, no transition)

**Task Row anatomy (left to right):**
1. Priority border: 3px left border
   - High: `--primary` (#6366f1 indigo)
   - Medium: transparent (no border)
   - Low: transparent (no border), entire row at `opacity: 0.6`
2. Completion checkbox: 16px circle, `--border` stroke. On click: task disappears instantly (`display: none`, no animation — PROJECT.md constraint)
3. Task title: 14px body text (`--foreground`)
   - High priority: `font-weight: 600` (semibold)
   - Medium priority: `font-weight: 400`
   - Low priority: `font-weight: 400`, row opacity 0.6
4. Overdue badge (right-aligned, shown only when due date < today):
   - Text: "N days ago" (12px/400)
   - Style: `--muted` background, `--muted-foreground` text, 4px border-radius, 4px/8px padding
5. Row tap target: entire row is clickable → navigates to Edit Task screen

**Empty state (shown when 0 active tasks):**
- Centered vertically and horizontally in the list area
- Heading: "All clear — nothing left for today." (18px/600, `--foreground`)
- Body: "Add a task above to get started." (14px/400, `--muted-foreground`)
- No illustration, no icon — text only

**Cap behavior (when tasks > 7):**
- Only 7 rows rendered; remaining tasks are silently suppressed in Phase 2
- No "show more" link in Phase 2

---

### Screen 2: Add Task

**Layout:** Centered form card on `--background`, max-width 480px, horizontal margin auto.

**Header:**
- Display text: "Add Task" (24px/600)
- Back navigation: text link "← Today" (14px/400, `--muted-foreground`) top-left

**Form fields (top to bottom, each with 16px vertical gap between fields):**

1. **Title input** (required)
   - Label: "Task" (12px/400, `--muted-foreground`)
   - Input: full-width, shadcn `<Input>` component, placeholder "What needs to get done?"
   - Validation: empty title blocks Save — inline error appears below input

2. **Due date picker** (optional)
   - Label: "Due date" (12px/400, `--muted-foreground`)
   - Control: shadcn `<Popover>` + `<Calendar>` — native date picker via Radix
   - Display value when unset: "No due date" (14px/400, `--muted-foreground`)
   - Display value when set: "Fri, Mar 21" (14px/400, `--foreground`)

3. **Priority selector** (required, defaults to Medium)
   - Label: "Priority" (12px/400, `--muted-foreground`)
   - Control: shadcn `<ToggleGroup>` with three options: "Low" | "Medium" | "High"
   - Default selection: Medium
   - Active state: `--primary` background, white label
   - Inactive state: `--muted` background, `--foreground` label

**Actions (bottom of form, 16px gap between buttons):**
- "Save Task" button: full-width, `--primary` background, white label, 14px/400 (primary CTA)
- No secondary cancel button — use "← Today" back link instead

---

### Screen 3: Edit Task

**Layout:** Identical to Add Task screen. Max-width 480px centered card.

**Header:**
- Display text: "Edit Task" (24px/600)
- Back navigation: "← Today" (14px/400, `--muted-foreground`) top-left

**Form fields:** Same as Add Task, all pre-filled with existing task values.

**Actions (bottom of form):**
- "Save Task" button: full-width, `--primary` background, white label (primary CTA)
- "Delete Task" button: full-width, `--destructive` background, white label, 14px/400
- Vertical stacking order: Save above Delete, 8px gap between them

**Delete confirmation:**
- Inline confirmation replaces Delete button on first click
- Replacement text: "Are you sure? This cannot be undone." (14px/400, `--destructive` color)
- Two inline options appear: "Yes, delete" (14px/600, `--destructive`) | "Cancel" (14px/400, `--muted-foreground`)
- No modal dialog — inline replacement only, no animation

## Interaction Contract

| Interaction | Behavior | Animation |
|-------------|----------|-----------|
| Check task complete | Instant `display: none` on row | None — PROJECT.md constraint |
| Click task row | Navigate to Edit Task screen | None |
| Click "+ Add Task" | Navigate to Add Task screen | None |
| Click "← Today" | Navigate back to Today view | None |
| Click "Save Task" (valid) | Persist task, navigate to Today view | None |
| Click "Save Task" (invalid title) | Show inline error "Task title is required." below input, no navigation | None |
| Click "Delete Task" (first click) | Replace button with inline confirmation copy | None |
| Click "Yes, delete" | Delete task, navigate to Today view | None |
| Click "Cancel" (delete) | Restore "Delete Task" button | None |
| OS theme change | CSS variables re-resolve automatically via `prefers-color-scheme` | None |

All interactions must respond in < 200ms (FOUND-06 constraint).
