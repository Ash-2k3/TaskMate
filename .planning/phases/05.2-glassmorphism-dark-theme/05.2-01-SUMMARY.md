## Decisions Made
- Used raw CSS `backdrop-filter: blur(16px)` in `@layer components` instead of `@apply backdrop-blur-*` Tailwind utility — avoids JIT purge ambiguity per RESEARCH.md recommendation
- Orbs use `filter: blur(80px)` on the orb element itself (blurs the orb outward), NOT `backdrop-filter` — this is correct per RESEARCH.md: backdrop-filter would need transparency in the orb's parent which creates the nested backdrop-filter Chromium limitation
- `--border` CSS variable kept at `240 20% 15%` (solid dark border for non-glass elements); glass surfaces use `rgba(255,255,255,0.10)` inline in `.glass` class directly — per RESEARCH.md Pitfall 2 note in PLAN.md
- Button gradient delivered via `.btn-gradient` CSS class added to `@layer components`, referenced from CVA `default` variant — keeps shadcn component system clean while overriding appearance
