### 1. Glass visual rendering

**Test:** Launch the app, inspect the Today screen.

**Why human:** `backdrop-filter` effect and `filter: blur(80px)` on orbs require Electron's Chromium renderer to evaluate.

### 2. Glass hover interactions

**Test:** Hover over each task row, then hover over the primary button.
**Expected:** Task row lifts 2px, background brightens slightly, border turns faint cyan. Button shows cyan glow shadow.
**Why human:** CSS transitions and hover state rendering require runtime.

### 3. Reflection notification system

**Test:** Trigger the reflection notification (set clock to after 21:00 with no existing reflection today).
**Expected:** A macOS system notification appears with title "TaskMate" and body "Time to reflect on your day 🌙" before the in-app reflection modal opens. Clicking the notification brings the window to focus.
**Why human:** Electron Notification API and scheduler timing require runtime.

### 4. DatePicker v9 calendar

**Test:** Open AddTask, click the due date DatePicker.
**Expected:** Calendar popover renders with a glass dark background, readable month/day headers, correct navigation buttons, and cyan highlight on the selected day. No console errors about unknown classNames.
**Why human:** react-day-picker v9 classNames rendering requires runtime.

### 5. Notification copy

**Test:** Let a task reminder fire (or inspect via scheduler logs).
**Expected:** Title reads "TaskMate" (not "TaskMate Reminder"); body has personal tone with emoji. Re-notification body contains "you've got this 💪". Weekly summary body contains "see how you did ✨".
**Why human:** Electron system notifications require runtime.
