<decisions>
## Implementation Decisions

### Dark base palette
- **D-01:** Force dark mode globally — add `class="dark"` to `<html>` element in `src/renderer/index.html`. Remove light-mode `:root` tokens; the `.dark` block becomes the only active theme.
- **D-02:** Base background `--background`: `240 33% 3%` (renders as `#080810` — very dark blue-black, matches portfolio)
- **D-03:** `--foreground`: `214 32% 96%` (renders as `#f1f5f9` — light slate)
- **D-04:** `--muted-foreground`: `215 20% 59%` (renders as `#94a3b8` — slate gray secondary text)
- **D-05:** `--card`: `240 25% 6%` — slightly elevated solid base for non-glass fallback contexts
- **D-06:** `--border`: transparent white — expressed as `0 0% 100% / 0.10` for the 10% white border pattern. Use `border border-white/10` utility in Tailwind for all borders.
- **D-07:** `--primary` (cyan): `189 94% 43%` (renders as `#06b6d4`)
- **D-08:** `--primary-foreground`: `0 0% 100%`
- **D-09:** `--ring`: `189 94% 43%` (cyan focus ring, matches primary)
- **D-10:** `--muted`: `240 20% 12%` — subtle muted surface
- **D-11:** `--accent`: `240 20% 12%` — same as muted for compatibility

### Glass surface system
- **D-12:** Glass surface CSS: `background: rgba(255, 255, 255, 0.05); backdrop-filter: blur(16px); -webkit-backdrop-filter: blur(16px); border: 1px solid rgba(255, 255, 255, 0.10);`
- **D-13:** Glass hover state: `background: rgba(255, 255, 255, 0.09); border-color: rgba(6, 182, 212, 0.25);` — subtle cyan border glow on hover
- **D-14:** Define `.glass` utility class in `index.css` under `@layer components` — apply to nav, cards, modals, popovers, and form wrappers
- **D-15:** Define `.glass-hover` modifier that adds the D-13 hover states with transition
- **D-16:** Border radius: `16px` (`rounded-2xl`) for main glass card surfaces; `8px` (`rounded-lg`) for smaller glass elements (rows, inputs, badges)

### Background orbs
- **D-17:** Two fixed-position orbs, `pointer-events-none`, `z-0`, rendered in `App.tsx` as siblings to the main content wrapper
- **D-18:** Orb 1 (cyan): `500px × 500px`, `background: radial-gradient(circle, rgba(6, 182, 212, 0.18) 0%, transparent 70%)`, positioned top-right (`top: -100px; right: -100px`)
- **D-19:** Orb 2 (indigo): `400px × 400px`, `background: radial-gradient(circle, rgba(99, 102, 241, 0.15) 0%, transparent 70%)`, positioned bottom-left (`bottom: -80px; left: -80px`)
- **D-20:** Orbs use `filter: blur(80px)` — soft diffuse glow, not sharp circles
- **D-21:** Main content wrapper wraps over orbs with `relative z-10` so content is always above

### Animations (lifted from Phase 5.1 no-animation constraint)
- **D-22:** Global transition token: `all 0.3s cubic-bezier(0.4, 0, 0.2, 1)` — apply via `.transition-glass` utility class in `index.css`
- **D-23:** Card/row hover lift: `transform: translateY(-2px)` — subtle, not jarring (portfolio uses -3px; desktop app feels better at -2px)
- **D-24:** Button hover: add `box-shadow: 0 0 20px rgba(6, 182, 212, 0.2)` cyan glow
- **D-25:** No `animate-*` keyframe animations (spinners, pulses, bounces) — only `transition` on hover/focus state changes
- **D-26:** TaskRow hover: apply `.glass-hover` + `translateY(-2px)` instead of current `hover:bg-muted/40`

### Gradient accent
- **D-27:** Primary gradient: `linear-gradient(135deg, #06b6d4, #6366f1)` — used on: "Add Task" button background, active nav indicator, gradient text spans (none currently, but available)
- **D-28:** Apply gradient to "Add Task" / "Save Task" / primary `Button` default variant via CSS override in `index.css`
- **D-29:** Active nav tab: replace `border-t-2 border-primary` with gradient bottom border `border-b-2` using a `border-image` or an absolutely-positioned `2px` gradient line

### Component-level glass application

- **D-32:** TodayView header: `.glass` treatment as a card header panel (`rounded-2xl mx-4 mt-4 mb-0 px-4 py-4`) — not full-bleed edge-to-edge anymore
- **D-33:** ReflectionModal `DialogContent`: `.glass` with stronger blur (20px) — modal should feel most elevated

<specifics>
## Specific Ideas

- Reference: https://www.ashwathkannan.com/ — exact palette source. Every design decision maps to a live implementation.
- Portfolio glassmorphism rule: solid dark base (#080810) + glass elevated surfaces — NOT everything glass (avoids visual mud)
- The cyan/indigo gradient is the identity color of this design system — it appears on primary actions and active states
- Orbs are not animated (no keyframe rotation/movement) — they are static blurred radial gradients, atmospheric not distracting
- Notification 👀 emoji is intentional — personal tone, not sterile
- Reflection notification 🌙 — night-time, introspective, matches the 9 PM trigger context
- TaskRow as individual glass cards (not separated by border-b) — each task lifts on hover, communicates "this is an interactive item"

</specifics>

## Existing Code Insights

### Reusable Assets
- `.glass` utility will cascade via `@layer components` — all components pick it up by class name, no per-file CSS changes
- CSS variable system (`--background`, `--card`, `--border`, etc.) is used consistently — token changes in `index.css` cascade everywhere automatically
- `bg-card` already applied to nav bar (Phase 5.1 D-18) — just swap to `.glass`

### Established Patterns
- shadcn Button `variant="default"` is the primary action button — override its background to the gradient in `index.css` under `@layer components`
- `border-border` is the standard border utility — supplement with `border-white/10` for glass surfaces
- `hover:bg-muted/40` on TaskRow — replace entirely with `.glass-hover` transition
- `bg-background` on task rows — remove; glass surface replaces it

### Integration Points
- `App.tsx` is the right place for background orbs — they must be behind all routed content; use `relative` wrapper + `fixed` orbs + `z-10` content
- Scheduler's reflection trigger (line 76-83 in reminder-scheduler.ts) — add `new Notification({...}).show()` immediately before `win.webContents.send('prompt:reflection')`
- `DialogContent` in ReflectionModal — add `.glass` to `className` prop
