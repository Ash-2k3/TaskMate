## Summary

This phase is a pure visual refactor — no new features, no new routes, no behavior changes. Every target file has been read and every change is precisely specified in the CONTEXT.md decisions. The changes reduce visual noise: removing filled row backgrounds, collapsing priority signals to weight only, unifying form screen layouts, and flattening the accordion card style in ReflectionsHistory. All changes stay within the existing Tailwind token system (`bg-background`, `text-muted-foreground`, `border-border`, etc.) — no new libraries, no new dependencies.

The project has one hard constraint that has already been observed in prior phases: **no animation or transition classes**. This applies equally here. The only live indigo touchpoints after this phase are the "Add Task" button (Button variant="default") and the active nav tab indicator (`text-primary border-t-2 border-primary`). Everything else goes neutral.

**Primary recommendation:** Execute changes file-by-file in dependency order — CSS variables first, then TaskRow (the most used component), then screens (TodayView, AddTask, EditTask, ReflectionsHistory), then ReflectionModal last (lowest-risk). Verify indigo leakage audit in App.tsx is clean before marking the phase done.

## Architecture Patterns

### Token-first class authoring

All Tailwind classes MUST use CSS variable tokens, not raw color values. This is an established project constraint.

Correct: `bg-background`, `text-muted-foreground`, `border-border`, `bg-muted/40`
Wrong: `bg-white`, `text-gray-500`, `border-gray-200`, `bg-slate-50`

The project has confirmed: `tailwind.config.js` already wires all token aliases — no changes needed there.

### Multi-class conditional pattern (existing in TaskRow)

TaskRow uses the array-filter-join pattern for conditional classes:

```typescript
className={[
  'base classes here',
  condition ? 'conditional-class' : '',
]
  .filter(Boolean)
  .join(' ')}
```

This is the established pattern. Continue using it for the title span's priority-based text color.

### No transition classes

Project constraint (PROJECT.md): do NOT add `transition-*`, `duration-*`, or `ease-*` classes anywhere. Hover state changes are instant — this is intentional.

## Common Pitfalls

### Pitfall 1: Leaving `opacity-60` on low-priority rows
**What goes wrong:** Removing `border-l-[3px] border-transparent` (D-12) and forgetting to also remove `opacity-60` (D-11). The opacity dims the checkbox and the overdue badge, making the entire row look broken rather than just the text muted.
**Why it happens:** D-11 and D-12 are separate decisions but both touch the same row element.
**How to avoid:** Remove `opacity-60` from the outer div class list AND add `text-muted-foreground` to the title span's conditional class for low priority.
**Warning sign:** Low priority tasks look faded but the checkbox is also grey.

### Pitfall 2: Missing the second and third `font-bold` instances in ReflectionsHistory
**What goes wrong:** The loading state and empty state at the top of ReflectionsHistory also use `font-bold`. Changing only the main render path leaves the loading/empty headings bold.
**Why it happens:** Three separate returns in the component, each rendering the `<h1>`.
**How to avoid:** Search for all `font-bold` in ReflectionsHistory.tsx — there are three instances (loading, empty, and main render).
**Warning sign:** Header looks bold when the history is loading or empty.

### Pitfall 3: Breaking accordion expand behavior when removing card wrapper
**What goes wrong:** The accordion in ReflectionsHistory currently uses `<div className="border rounded-lg">` as the container for both the toggle button and the expanded content. Removing it means the button and expanded section are now siblings, not parent/child.
**Why it happens:** The flat-row pattern changes structure, not just classes.
**How to avoid:** Each date entry becomes: `<div className="border-b border-border">` wrapping the button and expanded section — still needs a thin wrapper, just not a card-style one.
**Warning sign:** Expanded content floats outside its row or the bottom border appears between button and expanded content rather than after both.

### Pitfall 4: Accidentally breaking the yellow missed-reminders banner
**What goes wrong:** A sweep to remove non-standard border colors catches `border-yellow-300` in TodayView and removes it.
**Why it happens:** D-07 says remove non-`border-border` accent borders "except where semantically required" — but the exception is easy to miss during a broad search-and-replace.
**How to avoid:** The missed reminders banner (`border-yellow-300 bg-yellow-50`) in TodayView is intentional and must stay. Only touch TaskRow for border changes in TodayView.
**Warning sign:** Missed reminders warning loses its yellow styling.

### Pitfall 5: Removing centering from AddTask outer div but keeping inner max-w wrapper
**What goes wrong:** Removing `flex flex-col items-center` from the outer div but leaving `<div className="w-full max-w-[480px]">` as an inner wrapper. The form fields still cap at 480px because the max-w constraint is on the inner element.
**Why it happens:** Two-element centering pattern — both the flex-center on the outer AND the max-w on the inner must be removed.
**How to avoid:** Remove the inner `<div className="w-full max-w-[480px]">` wrapper entirely. The form fields (space-y-4 div and children) become direct children of the outer div.
**Warning sign:** Add Task form fields don't reach full viewport width.

## State of the Art

| Old Approach | Current Approach | Notes |
|--------------|------------------|-------|
| Colored left border for priority | Text weight only | Industry pattern (Linear, Craft): signal importance through typography weight, not color |
| `focus:ring-2` | `focus-visible:ring-1` | `focus-visible` avoids showing ring on mouse click; only shows on keyboard navigation. shadcn Input uses this. |
| Card-style rounded-lg rows | Flat border-b rows | List-style UX: rows feel like text on a page, not cards in a grid |
| `bg-muted` row fills | `bg-background` rows | Reduces visual noise; muted fills draw attention away from content |
