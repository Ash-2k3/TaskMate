# Phase 4.1: UI Polish — Minimalist Redesign - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Apply a minimalist visual redesign across all existing screens: TodayView, AddTask, EditTask, ReflectionsHistory, ReflectionModal, and the nav bar in App.tsx. No new features, no new routes, no behavior changes. Only visual/layout changes: task row style, color palette usage, priority signal, and form screen layout consistency.

</domain>

<decisions>
## Implementation Decisions

### Task row visual treatment
- **D-01:** Remove `bg-muted` fill from TaskRow — rows use `bg-background` (clean white) with `border-b border-border` separator only
- **D-02:** Hover state changes from `hover:bg-accent` to `hover:bg-muted/40` (lighter, more subtle)
- **D-03:** Left border used for high-priority no longer colored indigo — replaced with weight-only signal (see Priority below)
- **D-04:** Row padding stays `px-4 py-3` (tighten from current `py-4`)

### Color direction — near-monochrome
- **D-05:** Indigo accent (`--primary`) retained but usage reduced to exactly 2 touchpoints: (1) "Add Task" button, (2) active nav tab indicator
- **D-06:** All other current uses of `text-primary` or `border-primary` on non-nav/non-CTA elements are replaced with neutral equivalents (`text-foreground`, `border-border`)
- **D-07:** Border colors lightened — `border-slate-200` / `border-border` used consistently; no `border-yellow-300` style accent borders except where semantically required (missed reminders warning stays yellow — it's a semantic warning, not decoration)
- **D-08:** `--border` CSS variable adjusted to a lighter value (214.3 31.8% 91.4% → 214.3 20% 93%); or apply `opacity-70` to existing border tokens — Claude's discretion on exact value

### Priority signal (weight-only, no color)
- **D-09:** High priority: `font-semibold text-foreground` — bold text, no left border color
- **D-10:** Medium priority: `font-normal text-foreground` — unchanged
- **D-11:** Low priority: `font-normal text-muted-foreground` — faded text color, remove the `opacity-60` whole-row approach (opacity dims the checkbox and badge too, which looks unintentional)
- **D-12:** Remove `border-l-[3px] border-primary` and `border-l-[3px] border-transparent` from TaskRow entirely; left border gone

### Form screen layout (Add/Edit)
- **D-13:** AddTask and EditTask: remove `flex flex-col items-center` + `max-w-[480px]` wrapper — use `px-6 py-6` flush-left layout to match TodayView
- **D-14:** Form fields span full width (same as TodayView content width)
- **D-15:** Back button (`← Today`) and screen title stay at top, same `px-6 py-6` padding as TodayView header

### Nav bar
- **D-16:** Nav bar remains text-only (no icons) — active state stays `text-primary border-t-2 border-primary` (this is one of the 2 kept indigo touchpoints)
- **D-17:** Nav bar gets `bg-background border-t border-border` (already correct); no visual changes needed beyond what falls out of the color simplification

### Reflections history screen
- **D-18:** ReflectionsHistory header: replace `font-bold` with `font-semibold` (consistent with TodayView `text-2xl font-semibold`)
- **D-19:** Accordion rows: replace `border rounded-lg` cards with flat `border-b border-border` separator rows matching the new TaskRow style — no rounded borders, no card chrome
- **D-20:** Expanded answers: `px-4 pb-3 space-y-3` — same padding; question labels stay `text-sm text-muted-foreground`

### ReflectionModal
- **D-21:** No structural changes to ReflectionModal — it's a Dialog overlay and is already minimal
- **D-22:** Textarea class: replace `focus:ring-2 focus:ring-ring` with `focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-ring` for consistency with shadcn Input pattern

### Claude's Discretion
- Exact hover background value for task rows (somewhere between bg-transparent and bg-muted/30)
- Whether to tighten `py-6` header padding in TodayView to `py-5`
- Exact border opacity value for the lightened border tokens
- Whether EmptyState component needs visual updates to match (if current implementation looks out of place with new rows)

</decisions>

<specifics>
## Specific Ideas

- "Clean white + hairline dividers" — like Craft or Linear's list style. Rows should feel like text on a page, not cards in a list.
- Near-monochrome: indigo stays but only for interactive affordances (the button you click to add, the tab that's active). Content is pure neutral.
- Priority through weight, not color: bold text communicates importance without adding visual noise.
- Full-width forms: app should feel like one surface, not a task app + a modal-style form bolted on.

</specifics>
