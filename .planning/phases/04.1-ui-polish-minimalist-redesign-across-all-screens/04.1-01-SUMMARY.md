# Phase 04.1 Plan 01: CSS Border Token and TaskRow Minimalist Redesign Summary

**Lightened --border token (214.3 20% 93%) and TaskRow redesigned with clean bg-background rows, hairline dividers, font-weight-only priority signals, and subtle muted hover state**

## Accomplishments

- Lightened `--border` CSS variable from `214.3 31.8% 91.4%` to `214.3 20% 93%` — less blue-tinted, paler separator lines app-wide
- Redesigned TaskRow to render on `bg-background` (white) instead of `bg-muted` (grey fill), establishing clean list row aesthetic
- Replaced colored left-border priority system with font-weight signals: high = `font-semibold text-foreground`, low = `font-normal text-muted-foreground`
- Removed `opacity-60` whole-row dimming for low priority (now uses text color only)
- Replaced `hover:bg-accent` with `hover:bg-muted/40` for a subtler, less saturated hover wash

## Task Commits

Each task was committed atomically:

1. **Task 1: Lighten --border CSS variable in index.css** - `5f7178f` (feat)
2. **Task 2: Redesign TaskRow — clean background, weight-only priority, subtle hover** - `18eb062` (feat)

## Files Created/Modified

- `src/renderer/index.css` - Changed `:root` `--border` from `214.3 31.8% 91.4%` to `214.3 20% 93%`; dark mode unchanged
- `src/renderer/components/TaskRow.tsx` - Outer div uses `bg-background hover:bg-muted/40 py-3`; removed `border-l-[3px]` and `opacity-60`; title span priority classes updated

## Decisions Made

- Removed colored left borders: weight-only priority is sufficient for the minimalist aesthetic; colored borders were the most prominent "accent" leakage in the task list
- Low priority text-only signal: `text-muted-foreground` on title is less aggressive than `opacity-60` which affected the entire row including the checkbox
- `hover:bg-muted/40` chosen over `hover:bg-accent` — `bg-accent` is the same color as the old `bg-muted` fill, making it too saturated for a row that's already white
